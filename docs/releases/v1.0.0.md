# AI Studio v1.0.0 Release Notes

## Release Summary

AI Studio v1.0.0 is the portfolio-ready release of the local AIGC application platform. It consolidates AI Chat, Text to Image, Face Swap, unified Workflow History, SQLite persistence, and the Workspace Sidebar into a stable demonstration package.

## Highlights

- Unified Workspace UI with desktop Sidebar and mobile Sidebar drawer.
- AI Chat with role-based prompts and real SSE streaming through the LLM Gateway.
- Text to Image through ComfyUI workflows and image model registry.
- Face Swap through FaceFusion executor integration.
- Workflow History for image generation and face swap runs.
- SQLite persistence for workflow records across backend restarts.
- Local file storage for uploads and generated outputs.

## Architecture

The application keeps endpoints thin and routes workflow logic through service boundaries:

- Next.js frontend renders the workspace and workflow panels.
- FastAPI exposes REST and SSE endpoints.
- Services isolate Chat, Image Generation, Face Swap, Files, Tasks, and History.
- Runtime integrations are isolated behind clients, registries, or executors.
- SQLite stores workflow history metadata; local file storage keeps uploads and outputs.

## Validation Results

Release validation should include:

- Backend pytest: `python -m pytest -q`
- Frontend typecheck: `npm run typecheck`
- Frontend lint: `npm run lint`
- Frontend production build: `npm run build`
- Page checks for `/`, `/chat`, `/images`, `/face-swap`, `/history`, and history detail URLs.
- Manual runtime checks for Chat, Image Generation, and Face Swap when local runtimes are configured.

## Known Limitations

- ComfyUI, FaceFusion, and DeepSeek/Volcengine credentials are local runtime dependencies and are not bundled.
- Image generation model availability depends on the local ComfyUI model directory.
- Face Swap performance depends on the local FaceFusion runtime and execution provider.
- Chat requires a valid OpenAI-compatible DeepSeek endpoint and API key.
- The project is a local portfolio application and does not include authentication, multi-user isolation, or cloud deployment.

## Upgrade Notes

- Copy `.env.example` to `.env` and review runtime paths before starting the app.
- Existing SQLite history remains in `data/ai_studio.db` unless `AI_STUDIO_DATABASE_URL` is changed.
- Local upload and output files remain ignored by Git.

## Demo Checklist

1. Start backend and frontend with `./scripts/start-dev.sh`.
2. Open `http://127.0.0.1:3000`.
3. Confirm Sidebar navigation and runtime status.
4. Send one AI Chat message.
5. Generate one image if ComfyUI is running.
6. Run one Face Swap task if FaceFusion is configured.
7. Open Workflow History and inspect the new records.
8. Restart the backend and confirm History records are still visible.
